Beetle

High Availability Messaging with Redundant Queues

About

Beetle grew out of a project to improve an existing ActiveMQ based messaging infrastructure.

Usage

Publishing

    b = Beetle::Client.new
    b.register_queue("test")
    b.register_message("test")
    b.publish("test", "I'm a test message")

Subscribing

    b.register_handler("test") { |message| puts message.data }
    b.listen

Prerequisites

To set up a redundant messaging system you will need

  • at least 2 AMQP servers (we use RabbitMQ)
  • at least one Redis server

Gem Dependencies

At runtime, Beetle will use

  • uuid4r (which needs ossp-uuid)
  • bunny
  • redis-rb (which needs redis)
  • amqp (which is based on eventmachine)
  • activesupport

For development, you‘ll need

  • mocha
  • rcov
